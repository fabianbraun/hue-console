About the Hue Console

This is just a fun project to try out the Hue API in the Angular web environment and learn a little bit about color models. One thing to point out is that client sets color temperature. Most apps, including the official one from Philps, don't, and color temperature is very important for mood, psychology, and simply matching the decor, time of day, and other light sources in your home.

FAQ

Can I use this site from my phone or tablet?
Yes, but you'll need to host it locally somewhere on the same network as your hue bridge or emulator, such as your laptop or other small web-server.

Then why didn't you make it a phone app? :)
I may in the future -- this was just for fun, and if you have hue lights, you probably have a few spare laptops around :)

What is this "register/add user" button?
Hue has some built in security to prevent unauthorized use. It's nothing as sophisticated as OAuth, but it's reasonable for now. Aside from being on the same local network, the out-of-the-box Hue needs to be configured to work with each new client (official philips app, other apps, this app, etc...). To register your client app, you must press the 'link' button on the actual hue bridge device and then click the 'register/add user' button on the main page. This will register a client with the name specified in the 'Client-Id' field. 'newdeveloper' is the default client id for Hue API applications, so it may already be registered with your bridge. See http://www.developers.meethue.com/documentation/getting-started/ for details

Why do some text and areas on the page display with red strikethrough?
While you can set any parameter on Hue that you want (hue, saturation, color temperature, etc...), not all values apply at all times. For example, since color temperature only corresponds to whiteish light values with a reddish to blueish cast, many colors, such as a deep intense lime green, don't have a corresponding color temperature value, at least not on the Hue system. Therefore, if you set your light colors, the temperature value is shown with strikeout to show that is currently doesn't apply. It is possible to convert color temperature to an RGB value preview, but I'm not sure what this conversion formula is, and I have heard it is inherently inaccurate, so I don't currently support it. I could use the Hue system to help calculate HSB (and then derive RGB) from color temperature, but that would cause a fair amount of network lag, so I'd rather not support that either.
